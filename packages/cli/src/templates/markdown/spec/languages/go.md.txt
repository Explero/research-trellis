# Go 书写规范

## 默认规则

- 使用项目已有的 `gofmt`（Go 格式化）和检查配置；提交前保持受影响文件经过 `gofmt`（Go 格式化）；
- 包名简短、小写且表达领域，导出标识使用可读名称。不要在包名中重复目录含义；
- 错误必须向调用方返回或在边界处处理；包装错误时保留原始原因，便于使用 `errors.Is`（错误匹配）和 `errors.As`（错误提取）；
- 把 `context.Context`（请求上下文）作为需要取消、截止时间或请求范围的方法第一个参数，不把它存进长期结构体；
- 让零值可用，明确资源所有权。打开文件、连接或锁后，确认关闭与释放路径；
- 并发代码要说明所有权、退出条件和错误传播。不要通过共享可变状态或无退出的 goroutine（协程）隐藏工作。

## 测试与检查

- 使用项目已有的 `go test`（Go 测试）、`go vet`（Go 静态检查）和其他检查命令；
- 表驱动测试适合多个等价输入，但测试名称仍应说明行为；
- 修改并发、超时或资源释放时，补充失败、取消或泄漏路径验证。
