# 语言书写规范

> **作用**：按实际修改的语言选择一份最小规范，与通用代码规范一起使用。

## 选择规则

- 先读取项目已有的语言配置、格式化配置、静态检查配置和真实代码；
- 只把当前修改涉及的语言规范加入任务上下文，不要一次加载全部文件；
- 同一任务包含多种语言时，分别引用对应规范；公共要求仍以 `../guides/general-code-guidelines.md`（通用代码规范）为准；
- 工具只是建议基线。项目未配置某项工具时，不要擅自引入它或进行大面积格式化。

| 文件 | 适用场景 |
| --- | --- |
| `typescript-javascript.md` | TypeScript、JavaScript、Node.js 工具和前端代码 |
| `python.md` | 数据处理、训练、评估、自动化与科研脚本 |
| `go.md` | Go 服务、命令行工具和并发程序 |
| `rust.md` | Rust 工具、库和可靠性要求较高的组件 |
| `cpp.md` | C++ 性能模块、原生扩展和数值计算代码 |
| `shell.md` | Bash 脚本、实验编排和本地自动化 |

语言规范提供默认书写规则，不改变研究方案、数据边界、指标定义或任务关闭条件。
